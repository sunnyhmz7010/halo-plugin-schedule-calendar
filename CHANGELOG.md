# 更新日志

## v1.1.0-beta.4

- 优化控制台事项管理列表，移除循环信息的重复展示。
- 将时间、循环、本周展开、地点与备注收敛到标题下方同一行摘要区域。
- 调整事项摘要的自适应布局，长内容可自动换行，窄屏下显示更紧凑。

## v1.1.0-beta.3

- 新增 Halo 主题可直接调用的 Finder API：`scheduleCalendarFinder`。
- Finder 现支持 `week(start)`、`get(name)`、`listAll()` 三个方法，便于主题侧读取周视图和事项卡片数据。
- README 补充对外接口说明、Finder 用法和主题模板示例。

## v1.1.0-beta.2

- 修复控制台循环事项的可见性，周历色块增加循环标识，事项列表补充本周展开次数和下一次出现时间。
- 新增事项编辑能力，控制台现在支持直接修改已有事项的标题、时间、循环规则、颜色和说明。
- 补充前端循环工具测试，覆盖循环展开和循环说明格式化逻辑。

## v1.1.0-beta.1

- 新增循环事项能力，事项现在支持按每天、每周、每月、每年自动重复生成实例。
- 后台新增事项弹窗增加循环规则、循环间隔和截止日期配置项，周历和事项列表同步展示循环说明。
- 前台公开周历与编辑器日程卡片同步支持显示循环事项信息。
- 补充循环事项的查询展开与截止日期测试覆盖。

## v1.0.0

- 正式发布 Halo 日程日历插件 1.0.0。
- 收敛此前 beta 版本中的周历展示、事项管理、编辑器卡片和插件设置能力。
- 修复前后台周历在事项分列后可能被撑出宽度并触发额外滚动条的问题。

## v1.0.0-beta.5

- 基于当前最新提交重新发布测试版本。
- 延续 beta.4 的周历重叠布局、短事项显示、删除反馈和前后台文案整理修复。

## v1.0.0-beta.4

- 修复控制台与前台周历中重叠事项会互相覆盖的问题，改为按列分开展示。
- 优化短时长事项的色块显示策略，至少保留标题，并按空间显示时间和备注。
- 调整控制台新增事项、删除事项的交互反馈，弹框内直接显示创建错误，删除失败时回滚列表状态。
- 前台周历同步补充占用时间摘要，并统一星期与日期表头的居中样式。
- 精简前后台周视图中的重复范围和摘要文案，后台恢复“回到本周”按钮。
- 修复后台事项色块悬停信息中的地点文案，统一显示为“地点：xxx”。
- 删除事项时增加 Halo 原生 Toast 成功/失败提示。

## v1.0.0-beta.3

- 修复控制台日期选择框与上一周、下一周按钮高度不一致的问题。
- 修复新增事项弹框中的颜色选择器显示问题，改为可见色块和颜色值预览。
- 调整控制台页头与内容区间距，顶部保持原生页头，下方内容保持正常留白。
- 替换插件默认模板图标，改为自定义日历图标以满足发布要求。

## v1.0.0-beta.2

- 控制台与前台 `/schedule-calendar` 页面都增加上一周、下一周、回到本周和输入日期定位周的功能。
- 修复控制台周历左侧时间列与右侧星期表头未对齐的问题。
- 优化事项删除后的页面反馈，删除后无需手动刷新即可消失。
- 移除前台页面多余提示文案，收紧控制台页头为更原生的 Halo 风格。
- 优化新增事项颜色选择器显示。
- 补充控制台页和前台公开页的小屏设备适配。

## v1.0.0-beta.1

- 首次整理为统一测试发布版本。
