# 更新日志

## v1.0.0-beta.4

- 修复控制台与前台周历中重叠事项会互相覆盖的问题，改为按列分开展示。
- 优化短时长事项的色块显示策略，至少保留标题，并按空间显示时间和备注。
- 调整控制台新增事项、删除事项的交互反馈，弹框内直接显示创建错误，删除失败时回滚列表状态。
- 前台周历同步补充占用时间摘要，并统一星期与日期表头的居中样式。

## v1.0.0-beta.3

- 修复控制台日期选择框与上一周、下一周按钮高度不一致的问题。
- 修复新增事项弹框中的颜色选择器显示问题，改为可见色块和颜色值预览。
- 调整控制台页头与内容区间距，顶部保持原生页头，下方内容保持正常留白。
- 替换插件默认模板图标，改为自定义日历图标以满足发布要求。

## v1.0.0-beta.2

- 控制台与前台 `/schedule-calendar` 页面都增加上一周、下一周、回到本周和输入日期定位周的功能。
- 修复控制台周历左侧时间列与右侧星期表头未对齐的问题。
- 优化事项删除后的页面反馈，删除后无需手动刷新即可消失。
- 移除前台页面多余提示文案，收紧控制台页头为更原生的 Halo 风格。
- 优化新增事项颜色选择器显示。
- 补充控制台页和前台公开页的小屏设备适配。

## v1.0.0-beta.1

- 首次整理为统一测试发布版本。
- 控制台页面放置到系统工具入口下。
- 控制台界面改为 Halo 原生 UI 风格。
- 周视图改为大周历时间栅格，并使用色块展示事项。
- 调整控制台页面结构，新增事项改为弹窗录入，页面主体聚焦在周历和事项列表。
- 优化周概览信息展示，增加周范围、事项数和总占用摘要。
- 增加插件设置页，可配置前台路由页面标题。
- 修复插件设置扩展未被正确打包，确保 `schedule-calendar-settings` 可被正确加载。
- 修复控制台事项加载、创建相关问题。
- 修复前台 `/schedule-calendar` 页面 500 问题。
- 修复自定义模型未注册导致的 `Scheme not found` 问题。
- 修复插件启动测试，确保 CI 正常通过。
