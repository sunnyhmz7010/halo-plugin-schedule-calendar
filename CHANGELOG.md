# 更新日志

## v0.0.1-beta.4

- 修复控制台周历左侧时间列与右侧星期表头未对齐的问题。
- 控制台增加上一周、下一周和输入日期定位周的功能。
- 前台 `/schedule-calendar` 页面增加上一周、下一周和输入日期定位周的功能。

## v0.0.1-beta.3

- 调整控制台页面结构，改为更接近 Halo 官方插件的原生后台布局。
- 新增事项改为弹窗录入，页面主体聚焦在周历和事项列表。
- 优化周概览信息展示，增加周范围、事项数和总占用摘要。

## v0.0.1-beta.2

- 修复插件设置扩展未被正确打包，导致安装后提示 `setting extension was not found` 的问题。
- 将 `schedule-calendar-settings` 设置定义移动到插件扩展资源目录，确保 Halo 能正确加载设置页。

## v0.0.1-beta.1

- 首次整理为单一测试发布版本。
- 控制台页面放置到系统工具入口下。
- 控制台界面改为 Halo 原生 UI 风格。
- 周视图改为大周历时间栅格，并使用色块展示事项。
- 增加插件设置页，可配置前台路由页面标题。
- 修复控制台事项加载、创建相关问题。
- 修复前台 `/schedule-calendar` 页面 500 问题。
- 修复自定义模型未注册导致的 `Scheme not found` 问题。
- 修复插件启动测试，确保 CI 正常通过。
